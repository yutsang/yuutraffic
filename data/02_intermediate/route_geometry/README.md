# Route geometry storage (precomputed OSM routes)

Generated by `yuutraffic precompute`. One JSON file per route-direction (e.g. `KMB_1_0.json`, `CTB_930_1.json`).

## Format

Each file contains:

- **hash**: Stop sequence hash (for skip detection)
- **route_id**, **route_key**, **direction**, **company**
- **stops**: Array with stop metadata:
  - `stop_id` – API stop number
  - `stop_name`, `stop_name_tc` – English and Chinese names
  - `sequence` – Order along route
  - `company` – KMB/LWB, CTB, NWFB, etc.
  - `lat`, `lng`
  - `osm_node_sequence` – Index in OSM polyline where stop is snapped
  - `osm_node_name` – Human-readable label at that node
- **coords** – OSM polyline `[[lat,lng], ...]`

## Manual adjustment

Edit any file to fix route geometry. Change `coords` or `stops[].osm_node_sequence` as needed. No need to re-run precompute for edited routes.
